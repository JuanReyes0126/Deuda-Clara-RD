# Plan de implementación por fases

## Fase 1

- Definir arquitectura final
- Fijar estructura de carpetas
- Elegir estrategia de backend y despliegue

## Fase 2

- Ajustar scaffold de Next.js
- Configurar TypeScript estricto, ESLint y Prettier
- Preparar utilidades base, layout global y sistema de diseño

## Fase 3

- Diseñar schema Prisma
- Crear migración inicial
- Crear seed con admin, usuario demo, deudas, pagos y notificaciones

## Fase 4

- Registro
- Login
- Logout
- Recuperación y cambio de contraseña
- Perfil y onboarding

## Fase 5

- CRUD de deudas
- CRUD de pagos
- Reglas de recalculo de saldo
- Auditoría de cambios

## Fase 6

- Dashboard principal
- Widgets de deuda total, mínimos, urgencia, vencimientos y progreso
- Visualizaciones y gráficos

## Fase 7

- Motor de estrategias snowball, avalanche e híbrida
- Recomendaciones explicadas en lenguaje simple
- Simulaciones financieras clave

## Fase 8

- Notificaciones in-app
- Recordatorios por email
- Reportes y exportación PDF/CSV

## Fase 9

- Panel administrativo
- Gestión básica de usuarios
- Métricas y logs de auditoría
- Gestión de plantillas de email

## Fase 10

- Hardening
- Rate limiting
- Headers
- Validaciones finales
- Revisión de controles de seguridad

## Fase 11

- Unit tests
- Integration tests
- E2E críticos

## Fase 12

- README final
- SECURITY.md
- CHANGELOG
- Docker
- docker-compose
- GitHub Actions
- guía de despliegue
